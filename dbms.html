Database Management System (DBMS)

1. Data

Data refers to facts, concepts, or instructions represented in a structured format suitable for processing or communication by computers and humans. It is the foundational input used to generate information.

2. Data Types

  • Numeric
  • Character and String
  • Date and Time
  • Binary
  • Boolean
  • Special (e.g., JSON, XML, UUID)

3. Database

A database is an organized collection of data stored electronically, allowing efficient access, management, and manipulation. Used across many domains, databases are essential for decision-making and operations.

Types of Databases

  • Hierarchical
  • Network
  • Relational
  • Centralized
  • Distributed

4. Software Systems

Database Management System (DBMS)

DBMS is software that manages databases, allowing users to define, access, and manipulate data efficiently. Examples include Oracle, MySQL, MS Access, FoxPro, and DBase.

Other Systems Using Databases

  • Accounting Systems: Track financial data (e.g., QuickBooks, Tally)
  • Attendance Systems: Monitor presence and time (e.g., biometric, RFID)

5. Software Development

Database Design

Involves structuring tables, keys, and relationships to ensure data integrity and efficiency.

Database Administration

Focuses on managing database performance, backups, access control, and security.

6. Tools and Technologies

Server

Hosts the database and handles client transactions and requests.

SQL (Structured Query Language)

  • DDL: CREATE, ALTER, DROP
  • DML: INSERT, UPDATE, DELETE
  • DQL: SELECT
  • DCL: GRANT, REVOKE

SQL Joins: INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN

7. Key Concepts in Databases

CRUD Operations

  • Create
  • Read
  • Update
  • Delete

Database Keys

  • Primary Key
  • Foreign Key
  • Candidate Key
  • Composite Key

Constraints

  • NOT NULL
  • UNIQUE
  • PRIMARY KEY
  • FOREIGN KEY
  • CHECK
  • DEFAULT

Anomalies

  • Insertion Anomaly
  • Deletion Anomaly
  • Update Anomaly

Normalization

A process to reduce redundancy and enhance integrity by organizing data into normal forms:

  • 1NF – No repeating groups
  • 2NF – No partial dependencies
  • 3NF – No transitive dependencies
  • BCNF – Advanced form of 3NF

8. Database Security

Involves protecting data from unauthorized access and threats. Measures include:

  • User Authentication
  • Access Control
  • Encryption
  • Audit Trails
